Read or write exodus files loaded into memory

This is a convenience class wrapped around the exodus class to enable easier manipulation of exodus files that fit entirely into memory

Parameters

  • exo (exodus) ‚Äì An exodus object, if you want to load an exodus file into memory, or None if you want to create an empty exodus file.

  • variables (iterable) ‚Äì A list of variable names that are loaded into memory. Default is to load all variables

  • timesteps (iterable) ‚Äì A list of timestep indices that are loaded into memory. Default is to load all timesteps.

  • blocks (iterable) ‚Äì A list of block ids that are loaded into memory. Default is to load all blocks.

Gets the node indices and element connectivity of surface elements

This function ‚Äúskins‚Äù the element block, returning a list of node indices and a surface connectivity matrix.

Parameters

  • block_id (int) ‚Äì The ID number of the block that will be skinned.

  • keep_midside_nodes (bool) ‚Äì Specifies whether or not to keep midside nodes in the surface mesh. Default is False.

  • warn (bool) ‚Äì Specifies whether or not to warn the user if the block ID doesn‚Äôt have a skinning method defined for its element type. Default is True.

Returns

element_block_information ‚Äì A list of tuples of element information. These data are element_type, node_indices, block_surface_connectivity, and block_surface_original_elements. The element_type is a string representing the new block element type (‚Äòquad4‚Äô,‚Äôtri3‚Äô,etc.). The node_indices can be used as an index into the coordinate or nodal variable arrays to select nodes corresponding to this block. The block_surface_connectivity represents the connectivity array of the surface faces of the block. Values in this array correspond to indices into the node_indices array. To recover the connectivity array in the original node indices of the exodus file, it can be passed through the node_indices as node_indices[block_surface_connectivity]. The block_surface_original_elements array shows the original element indices of the block that each surface came from. This can be used to map element variables to the new surface mesh. This list will normally be length 1 unless an element type is processed that has two different surface elements in it (e.g. wedges have tris and quads)
